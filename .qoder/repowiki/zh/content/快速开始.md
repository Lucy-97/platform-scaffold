## 简介

Platform Scaffold 是一个现代化的微服务项目脚手架工具，基于经过验证的最佳实践构建。它能够一次性生成完整的微服务骨架，包括 Go 网关、Go API、Python AI 引擎、Next.js 前端和 React 后台管理界面，以及完整的部署配置。

该工具的核心优势在于：
- **一体化生成**：一条命令生成全套项目骨架
- **最佳实践集成**：内置成熟的架构设计和安全策略
- **高度可定制**：支持交互式配置和模块化选择
- **自包含二进制**：无需额外依赖即可运行

## 环境要求

在开始使用 Platform Scaffold 之前，请确保您的系统满足以下要求：

### 基础环境
- **Go 版本**：1.22 或更高版本
- **操作系统**：支持 macOS、Linux 和 Windows

### 开发工具
- **Git**：用于版本控制（可选）
- **文本编辑器**：支持 Go、TypeScript、Python 语法高亮
- **终端**：支持 Bash 或 PowerShell

### 可选但推荐的工具
- **Docker Compose**：用于本地开发环境
- **Node.js**：用于前端开发（Next.js/Vite）
- **Python 3.11+**：用于 AI 引擎开发

## 安装步骤

### 方法一：使用 Go Install（推荐）

最简单的方式是通过 Go 的官方安装命令：

```bash
go install github.com/platform-scaffold/cli/cmd/platform@latest
```

验证安装是否成功：

```bash
platform version
```

### 方法二：从源码编译

如果您需要最新开发版本或进行自定义修改：

```bash
# 克隆仓库
git clone https://github.com/platform-scaffold/cli.git
cd cli

# 编译二进制文件
go build ./cmd/platform

# 将可执行文件移动到系统路径
mv platform /usr/local/bin/
```

### 验证安装

安装完成后，您可以使用以下命令验证：

```bash
# 查看帮助信息
platform --help

# 查看版本
platform version
```

## 基本配置

### 交互式配置流程

首次使用时，Platform Scaffold 会引导您完成交互式配置过程。以下是所有可用的配置选项：

#### 项目基本信息
- **项目名 (ProjectName)**：采用 kebab-case 格式（如 `my-app`）
- **品牌名 (Brand)**：用于展示的名称（如 `MyApp`）
- **域名 (Domain)**：服务域名（如 `myapp.ai`）
- **Go Module 路径**：Go 模块导入路径（如 `github.com/me/my-app`）

#### 端口配置
- **Gateway 端口**：默认 8080
- **API 端口**：默认 8001
- **AI Engine 端口**：默认 8002
- **Web 端口**：默认 3000
- **Admin 端口**：默认 5174

#### 功能模块选择
- **AI Engine**：Python FastAPI 服务（默认启用）
- **Web 前端**：Next.js 应用（默认启用）
- **Admin 后台**：Vite + React 应用（默认启用）
- **Core Lib**：公共组件库（默认启用）

#### 其他选项
- **初始化 Git 仓库**：是否在生成后初始化 Git（默认启用）

### 非交互模式

对于自动化场景，可以使用 `--yes` 参数跳过交互：

```bash
platform init my-app --yes
```

注意：使用非交互模式时必须显式指定项目名。

## 命令行工具使用方法

### 主要命令

#### platform init
初始化新的微服务平台项目：

```bash
# 基本用法
platform init [项目名]

# 非交互模式
platform init my-app --yes

# 指定输出目录
platform init my-app -o /path/to/output
```

#### platform version
显示当前版本信息：

```bash
platform version
```

### 命令选项详解

#### init 子命令选项
- `--yes` 或 `-y`：使用默认值，跳过交互式提问
- `--output` 或 `-o`：指定输出目录（默认与项目名相同）

#### start.sh 脚本选项
生成的项目包含一个强大的启动脚本，支持多种操作：

```bash
# 启动所有服务
./deploy/local/start.sh start

# 启动后端服务（网关、API、AI引擎）
./deploy/local/start.sh start backend

# 启动特定服务
./deploy/local/start.sh start gateway
./deploy/local/start.sh start api
./deploy/local/start.sh start ai-engine
./deploy/local/start.sh start web
./deploy/local/start.sh start admin

# 停止服务
./deploy/local/start.sh stop all

# 查看状态
./deploy/local/start.sh status

# 查看日志
./deploy/local/start.sh logs gateway
```

## 常见问题解决

### 问题 1：端口冲突

**症状**：启动时提示端口已被占用

**解决方案**：
```bash
# 修改配置中的端口号
# 在交互式配置中修改端口
# 或在生成后手动修改配置文件
```

### 问题 2：权限不足

**症状**：无法创建文件或目录

**解决方案**：
```bash
# 检查目录权限
ls -la

# 修改目录权限
chmod 755 /path/to/target/directory

# 以管理员权限运行（不推荐）
sudo platform init my-app
```

### 问题 3：模板渲染错误

**症状**：生成过程中出现模板相关错误

**解决方案**：
```bash
# 检查 Go 版本兼容性
go version

# 更新到最新版本
go install github.com/platform-scaffold/cli/cmd/platform@latest

# 清理缓存重新安装
go clean -cache
go install github.com/platform-scaffold/cli/cmd/platform@latest
```

### 问题 4：网络连接问题

**症状**：下载依赖或模板时失败

**解决方案**：
```bash
# 检查网络连接
ping github.com

# 配置代理（如果需要）
export HTTP_PROXY=http://proxy-server:port
export HTTPS_PROXY=https://proxy-server:port

# 重新安装
go install github.com/platform-scaffold/cli/cmd/platform@latest
```

### 问题 5：启动脚本无法执行

**症状**：./deploy/local/start.sh: Permission denied

**解决方案**：
```bash
# 给予执行权限
chmod +x deploy/local/start.sh

# 或者使用 bash 直接执行
bash deploy/local/start.sh start
```

## 项目结构概览

### 生成后的项目结构

使用 Platform Scaffold 生成的项目具有清晰的层次结构：

```
my-app/
├── backend-gateway/      # Go Gin 网关（JWT/限流/CORS/proxy）
├── backend-api/          # Go API（handler→service→repository）
├── backend-ai-engine/    # Python FastAPI（AI 编排，只读）
├── frontend-web/         # Next.js 15 App Router
├── frontend-admin/       # Vite + React 19 后台
├── pkg-platform-core/    # 通用组件库（errcode/lock/cache/crypto/middleware）
├── deploy/
│   ├── local/           # docker-compose + start.sh
│   └── k3s/             # K3s manifests + 部署脚本
├── database/            # 数据库初始化脚本
├── README.md            # 项目文档
├── CLAUDE.md           # AI 辅助文档
├── .gitignore          # Git 忽略规则
└── .env.example        # 环境变量示例
```

### 核心架构约定

生成的项目遵循以下核心约定：

#### 写权限矩阵
- 所有数据库写操作只在 Go API 进程中执行
- Python AI Engine 仅限只读访问

#### 服务间鉴权
- Gateway → 下游服务使用 `X-Internal-Secret` 头
- 采用 constant-time 校验防止时序攻击

#### JWT 处理
- 网关负责 JWT 解析和验证
- 业务 API 通过 `X-User-UUID` 头识别用户身份

#### 配置管理
- 三层配置：编译期常量 → 启动期环境变量 → 运行期 `system_config`
- 敏感值使用 AES-256-GCM 加密存储

#### 错误处理
- 6 位业务码注册表
- HTTP 状态码仅用于基础设施层
