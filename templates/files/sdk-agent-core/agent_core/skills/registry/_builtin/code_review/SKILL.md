# 代码审查技能

## 审查流程

1. **代码风格**：检查命名规范、缩进、注释覆盖率
2. **安全性**：检查输入验证、SQL注入、XSS、硬编码凭证
3. **性能**：识别 N+1 查询、不必要的循环、内存泄漏风险
4. **可维护性**：DRY 原则、函数长度、圈复杂度

## 输出格式

```
[严重程度] 文件:行号 — 问题描述
  建议: 修复方案
```

严重程度：🔴 Critical / 🟡 Warning / 🔵 Info
